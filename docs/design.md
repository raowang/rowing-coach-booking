## Context

赛艇中心现有会员预约采用电话或前台登记的传统方式，流程繁琐、效率低下。会员与教练之间缺乏持续互动，练后反馈缺失，导致训练效果难以提升，会员粘性不足。

**当前痛点：**
- 预约流程僵化，无法根据会员状态动态调整
- 教练资源分配不均，热门教练预约困难
- 练后无反馈闭环，会员进步缓慢
- 关怀缺失，会员流失率高

**约束与假设：**
- 微信小程序为主要前端载体（会员端）
- 教练端使用企业微信或小程序
- AI能力基于本地推理（Apple Silicon M5 Max）+ 云端API混合实现
- 需要与现有会员管理系统对接

## Goals / Non-Goals

**Goals:**
- 构建以AI Agent为核心的智能预约系统，实现自然语言驱动的用户体验
- 建立完整的练后反馈闭环，AI持续学习优化教练推荐
- 实现个性化关怀，提升会员满意度和续卡率
- 降低前台预约工作量，提升教练日程利用率
- 通过本地推理大幅降低AI服务成本

**Non-Goals:**
- 不替换现有会员管理系统（对接而非重建）
- 不实现支付功能（第一期）
- 不支持多场馆管理（单一场馆）

## Decisions

### 1. AI Agent架构：双Agent模式

**决策：** 采用用户Agent + 教练Agent的双Agent架构
- 用户Agent：面向会员，处理预约、查询、反馈收集等对话交互
- 教练Agent：面向教练，处理日程确认、点评提交、建议录入

**替代方案：** 单一大模型统一处理 → **选择双Agent** 因为教练和用户场景差异大，角色分离更清晰

### 2. 对话框架：RAG + Memory + 本地推理

**决策：** 使用RAG（检索增强生成） + 会话Memory + 本地LLM实现上下文理解
- RAG：挂载业务知识库（教练信息、课程内容、场馆规则），向量存储于本地Qdrant
- Memory：用户会话历史 + 长期偏好画像，存储于PostgreSQL（Directus）
- **本地LLM**：Llama 3.1 8B处理简单对话（预约、查询、问候）
- **云端LLM**：复杂推理（训练建议生成、会员画像分析）调用腾讯混元/阿里通义

**替代方案：** 纯云端API → **选择本地+云端混合** 因为80%简单对话本地处理可节省70% API成本

### 3. AI推理架构：本地优先

**决策：** ASR、Embedding、ML、简单LLM全部本地运行于Apple Silicon M5 Max

| 组件 | 本地模型 | 规格 | 用途 |
|------|----------|------|------|
| ASR | Whisper Large V3 / SenseVoice | Q5 (~10GB) | 语音转文字 |
| Embedding | bge-m3 | Q8 (~3GB) | 向量生成 |
| 简单LLM | Llama 3.1 8B Q4 | (~5GB) | 简单对话、预约 |
| 复杂LLM | Llama 3.1 70B Q4 | (~40GB) | 复杂推理 |
| 向量库 | Qdrant (本地) | - | RAG存储 |

**推理框架：** ollama (统一管理) + llama.cpp (Metal GPU加速)

**替代方案：** 纯云端API → **选择本地推理** 因为长期成本降低70%+，且数据完全私有化

### 4. 教练推荐算法：规则 + 本地ML混合

**决策：** 规则引擎做初筛，本地ML模型做排序
- 规则：新手匹配有耐心的教练、进阶会员匹配专业教练等
- ML：基于历史预约数据学习会员-教练匹配偏好（本地训练）

**替代方案：** 纯ML推荐 → **选择混合** 因为冷启动时ML效果差，且规则可解释性强

### 5. 日程管理：状态机驱动

**决策：** 预约状态机：待确认 → 已确认 → 进行中 → 已完成 → 已评价
- 状态变更触发微信通知
- 超时自动提醒或释放名额

### 6. 数据存储：PostgreSQL + Directus + Valkey

**决策：** PostgreSQL（Directus托管）存储结构化数据，Valkey缓存会话状态和热门教练
- PostgreSQL + Directus：会员、教练、日程、训练记录等结构化数据，Directus 提供 REST/GraphQL API 并集成 MCP 服务供 Agent 调用
- Valkey：会话状态、热门教练缓存、Token 缓存

## Risks / Trade-offs

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 本地服务器宕机 | 高 | VPN远程重启 + 云端降级API |
| LLM响应延迟（本地70B） | 中 | 前端加载动画 + 流式输出 + 8B降级 |
| 语音识别准确率 | 中 | 文字输入作为备选，语音转文字后让用户确认 |
| 教练不配合使用系统 | 高 | 简化操作流程 + 激励措施（提升曝光） |
| 会员画像冷启动 | 中 | 初期问卷收集 + 快速学习机制 |
| 微信通知送达率 | 低 | 短信备用通知 + 小程序内消息 |

## Migration Plan

**Phase 1（MVP）：**
1. 搭建基础小程序框架 + AI对话界面
2. 部署本地推理服务（Whisper + bge-m3 + Llama 8B）
3. 实现教练录入和日程管理
4. 实现AI预约推荐（规则版本）
5. 微信通知集成

**Phase 2（智能化）：**
1. 部署Llama 70B处理复杂推理
2. AI教练点评收集
3. 训练建议生成
4. 会员画像学习

**Phase 3（闭环）：**
1. 完整反馈闭环
2. 个性化关怀系统
3. 高级推荐算法
4. 考虑部署到赛艇中心本地（Mac Studio）
